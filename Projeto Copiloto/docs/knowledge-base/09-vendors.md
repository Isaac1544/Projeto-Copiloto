# KB 09 — Vendors e Tecnologias Relevantes

## Objetivo

Registrar tecnologias relevantes para o domínio do Copiloto, sem assumir que todas estejam presentes em todos os clientes ou processos internos.

## Critério de inclusão

Uma tecnologia aparece aqui quando:

- é citada no briefing;
- aparece em informações públicas da Clear IT;
- é comum em operações de infraestrutura, cloud, service desk e suporte corporativo;
- será útil para orientar futuras bases de solução.

## Nutanix

### Conceitos úteis
- AHV / Acropolis Hypervisor.
- Prism.
- CVM (*Controller VM*).
- Clusters hiperconvergentes.

### Aplicação no projeto
O briefing cita exemplos relacionados a Acropolis. Portanto, Nutanix deve ser considerada tecnologia relevante para exemplos e KBs futuras.

## Microsoft

### Conceitos úteis
- Windows Server.
- Active Directory.
- DNS.
- DHCP.
- Hyper-V.
- Azure.

### Aplicação no projeto
Ambientes Microsoft são comuns em operações corporativas e podem aparecer em tickets de suporte.

## VMware

### Conceitos úteis
- ESXi.
- vCenter.
- Máquinas virtuais.
- Datastores.
- Clusters.

## Linux

### Conceitos úteis
- systemd.
- journalctl.
- SSH.
- permissões.
- serviços.
- logs em `/var/log`.

## Kubernetes

### Conceitos úteis
- Pods.
- Nodes.
- Deployments.
- Events.
- Namespaces.
- Logs de containers.

## Cloud

### Tecnologias possíveis
- AWS.
- Azure.
- Oracle Cloud.

## Observabilidade

### Conceitos úteis
- Métricas.
- Logs.
- Traces.
- Alertas.
- Correlação de eventos.

## Limitações

Este documento não afirma que uma tecnologia específica faz parte de um contrato, cliente ou operação interna da Clear IT. Ele apenas organiza tecnologias relevantes para o domínio do desafio.

## Fontes

- Briefing Desafio B.
- Informações públicas da Clear IT.
- Documentações públicas dos fabricantes.
